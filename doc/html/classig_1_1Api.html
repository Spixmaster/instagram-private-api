Detailed Description

This class represents the Instagram API which interacts with the Instagram servers.

Note
The cookies need to be updated whenever an endpoint is called as even then cookies are set.
Every function that calls post_req_check cannot be "noexcept".
Every "feed/timeline/" call needs to set the most recent seen media id.

std::string ig::Api::get_media_comments ( const std::string &  media_id,
const std::string &  max_id = ""

Gets the comments of the Instagram media.

Only a part of all comments is shown.

Normally, the function is called without max_id.

If there are more possible comments to show, the field exists: ""has_more_comments":true".

The following attribute will be set:

"next_max_id":"{\"server_cursor": "QVFDNHZuUmxaTFJzeHJSQ0UyejVwcGFpXzBOWGVHQXpfeTNhalVOS1lUTUFrRWZuaHBoakFBbXlVWEV1ejE4aWFGLXZPS29TZUI4c3F5dXl1RHZULVZ2Wg==","is_server_cursor_inverse": false}"

Use that value for max_id to receive the next page of comments.

Note
The pagination does not work like: 1, 2, 3,...
Parameters
[in]media_idThe proper media id.
[in]max_idFor pagination.
Returns
The server response.
